Customising NewsAgent for Internationalisation

Internationalisation

NewsAgent is designed to be system and language independent (with the obvious exception of the user documentation). However, I am not a linguist, so the provision of foreign language support with the current release of NewsAgent is somewhat limited.

Nevertheless, NewsAgent supports internationalisation, and it is possible for the user to edit NewsAgent's string tables and convert it into another language.

At the moment, some basic translations are provided, for instance Organisation becomes Organization if the program is being run using Java in the US.

Digging Into the Jar

All of NewsAgent's strings are stored in some files in the newsagent.jar file distributed with NewsAgent. This file is really just a PKZip file: you can decompress it with pkunzip orWinzip.

Inside the JAR file, you will notice two files called Strings.properties and Menus.properties. The Strings file contains all strings used in dialogue boxes and window titles, and the Menus file contains the order, layout and labels for menus.

To Internationalise NewsAgent, you should not edit these files directly. Instead, make a copy of them, renaming them to Strings_XX.properties and Menus_XX.properties, where XX is the two letter country code of your locale, for instance this is uk for Great Britain, us for the United States, fr for France, de for Germany and so on.

You can now edit these files and translate them as you wish, add them to the Jar file, and NewsAgent will automatically use your translations (provided your computer is set up properly for your locale).

Other Customisations

As mentioned above, you can also change the order and layout of menus and toolbars in NewsAgent by editing the Menus.properties file. You can even change the images NewsAgent uses for its toolbars by adding or replacing files in the images subdirectory of the JAR file.

More information on customising the toolbar is available in the Toolbar topic in Menus & Dialogues.
