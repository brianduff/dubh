Editing Preference Files

How NewsAgent Stores Your Preferences

The preferences system in NewsAgent uses a standard Java practice of storing preferences in a file with the extension properties, and keeping this file in your "home directory". The main preferences file for NewsAgent is user.properties. NewsAgent stores this in a subdirectory of your home, called .newsagent. The definition of "home directory" differs from system to system. Some examples of where to find the file are given below:

  • UNIX
    ~/.newsagent/user.properties
  • Windows 95
     C:\Program Files\JDK\.newsagent\user.properties
  • Windows NT (no home directory)
    C:\.newsagent\user.properties
  • Windows NT (home directory in profiles)
    C:\WINNT\Profiles\Cairndubh\.newsagent\user.properties

If you open this file in any text editor, you will see that each line is of the form:

newsagent.something.somethingelse=Something, possibly with spaces in it

This line sets the property newsagent.something.somethingelse (the left hand side) to the value (on the right hand side). The property (left hand side) is case sensitive. The NewsAgent preferences aren't consistent in their capitalisation, so check the property keys carefully.

A Cautionary Note

There are absolutely no guarantees as to how NewsAgent will behave if you set some of its properties to unusual values. In most cases, there is a reasonable degree of bounds checking, but it's always possible to find a case which might cause problems. You should therefore make a backup of your properties file before editing it by hand.

If the worst happens, and you break NewsAgent, just delete the properties file and NewsAgent will revert to its defaults.

Editing the properties file while NewsAgent is running is pointless, as NewsAgent will more than likely overwrite your changes.

The order of properties in the preference file is completely arbitrary, and will change whenever NewsAgent writes your preference file.

NewsAgent Properties

NEW: There are a number of new properties in version 1.02, and some preferences have been removed.

The following properties are available in this version of NewsAgent. You may find properties other than these in the properties file, such properties should be considered as non-finalised and subject to change in future versions.

newsagent.agents.send.active
A (space separated) list of full classnames of all Send Agents that are switched on, in the order they will be executed when a message is posted. Normally configured from the Send Agents Panel of the Preferences Dialogue.
newsagent.agents.send.alwayspreview
Send Agents are allowed to modify your outgoing message, for example add extra headers or change the message text in some way. By default, if a Send Agent has modified your message in some way, the message will be displayed to you before it is sent, and you will be asked to confirm that you really want to send the message. You can disable this preview by setting the newsagent.agent.send.alwayspreview preference to no. There is also a checkbox on the preview dialogue box that you can uncheck to prevent future previews. If you want to stop Send Agents from modifying your messages, you should use the Send Agents Panel of the Preferences Dialogue to disable offending Agents. Acceptible values = yes or no. Default value = yes.
newsagent.agents.send.ignoreerrors [NEW Since 1.02]
Send Agents can warn you about problems with your outgoing messages. If a Send Agent considers a problem to be particularly severe, it will display an error message and you will not be able to post the message until you have fixed the error. You can disable this behaviour by setting the newsagent.agent.send.ignoreerrors property to yes. Send Agents will no longer display any errors. Acceptible values = yes or no. Default value = no.
newsagent.agents.send.ignorewarnings [NEW Since 1.02]
Send Agents can warn you about problems with your outgoing messages. If you try to post a message that has warnings, the warnings will be displayed and you will be asked if you really want to send the message. If you set the newsagent.agent.send.ignorewarnings property to yes, all Send Agent warnings will be suppressed, and you will never be asked to confirm that you really want to post a message. Acceptible values = yes or no. Default value = no.
newsagent.agents.send.installed
A (space separated) list of full classnames of Send Agents that the user has installed manually. Normally configured from the Send Agents Panel of the Preferences Dialogue.
newsagent.agents.list.active
A (space separated) list of full classnames of all List Agents that are switched on, in the order they will be executed when a message is read. Normally configured from the List Agents Panel of the Preferences Dialogue.
newsagent.agents.list.installed
A (space separated) list of full classnames of List Agents that the user has installed manually. Normally configured from the List Agents Panel of the Preferences Dialogue.
newsagent.composer.AdditionalHeaders [NEW Since 1.02]
The message composer contains the two headers "Newsgroups" and "Subject" by default. Using the popup menu over the headers area, you can add and remove additional headers. This property contains the names of all additional headers you are using, separated by spaces.
newsagent.debug.Console [NEW Since 1.02]
Whether to use a windowed console for debugging output produced if you have the DebugMessages or ServerDump properties set (yes or no). This is provided for users who don't have a command line to read debugging output from (e.g. in NewsAgent v1.02 under Windows 32bit with the launcher .exe). You can also use the command line flag -console to enable the debugging console windows.
newsagent.debug.DebugMessages [Updated Since 1.02]
Whether to produce debug output (yes or no). You can also use the command line flag -debug to enable debug output.
newsagent.debug.ServerDump [Updated Since 1.02]
Whether to dump all status and command I/O to and from News Servers to the console (yes or no). May or may not be enabled in a release version. You can also use the command line flag -debugnet to enable network debugging.
newsagent.general.AutoUpdate
Whether to update the list of messages in a newsgroup periodicaly (yes or no). Normally configurable from the General Panel of the Preferences Dialogue.
newsagent.general.HelpBrowserApp [NEW Since 1.02]
The path to your favourite WWW browser. If this property is set, an external browser of your choice is always used for help. The property should be of the form "path/to/command {0}". {0} is replaced by the path to the HTML file that is being displayed. For instance "C:\progra~1\plus!\micros~1\iexplore.exe {0}". If this preference is not set (default), the internal browser is used.
newsagent.general.NewsgroupNotify
Whether to check for new newsgroups whenever you start NewsAgent (yes or no). Normally configurable from the General Panel of the Preferences Dialogue.
newsagent.general.UpdateInterval
The time in minutes between updates of the list of messages in a newsgroup. Normally configurable from theGeneral Panel of the Preferences Dialogue.
newsagent.identity.email
Your email address. Normally configured with the Identity Panel of the Preferences Dialogue.
newsagent.identity.organisation
Your organisation. Normally configured with theIdentity Panel of the Preferences Dialogue.
newsagent.identity.realname
Your real name. Normally configured with theIdentity Panel of the Preferences Dialogue.
newsagent.msgviewer.Headers [NEW Since 1.02]
You can now customise the headers that are displayed in the message display panel of the main NewsAgent window using this property. The property should be set to a space separated ordered list of header field names. (e.g. "From Subject Newsgroups Message-Id"). Message headers are case insensitive.
newsagent.send.AddNewsAgentHeaders [NEW Since 1.02]
NewsAgent adds the two headers X-Mailer and X-Mailer-Url to all outgoing messages by default. Setting this property to no will disable this behaviour.
newsagent.send.HardBreaks [NEW Since 1.02]
Some Newsreaders use message viewers that don't wrap text to word boundaries. To ensure that messages appear correctly in such applications, this option ensures there is a carriage return at the end of each line of text in your outgoing messages. If this option is switched off, each paragraph is sent as a line of unbroken text, which will be formatted as the recieving newsreader sees fit. Default value = yes. Acceptable values = yes or no.
newsagent.send.IncludeBehaviour [NEW Since 1.02]
Determines whether and how NewsAgent inserts quoted text from the original message into replies. Can be one of the following three values (case insensitive): all - quote the entire post in the reply (default), selected - quote only text that is currently selected in the message viewer window in the reply, none - do not quote from the original post.
newsagent.send.IncludeHeading [NEW Since 1.02]
The text to add at the start of quoted text in reply messages. Use the wildcard {header-name} to insert any header from the original message. Header field names are case insensitive. The default value is "In message {message-id}, {x-na-realname} wrote:". The special field "x-na-realname" inserts just the real name of the sender, extracted from the From header
newsagent.send.IncludeMessages [deprecated since 1.02]
Whether to include the original message in replies (yes or no). Normally configurable from the Send Panel of the Preferences Dialogue. This preference has been deprecated. It will still work, but you should use the newsagent.send.IncludeBehaviour preference instead. If you have newsagent.send.IncludeMessages = yes, it will behave like newsagent.send.IncludeBehaviour=all.
newsagent.send.IncludePrefix [NEW Since 1.02]
The character(s) to prefix included messages with. The default (and recommended) value is "> ". Bear in mind that long prefix Strings might cause lines to wrap badly on some Newsreaders.
newsagent.send.StorePosts
Whether to store copies of the user's messages in a special folder (yes or no). Normally configurable from the Send Panel of the Preferences Dialogue.
newsagent.servers.SMTPHostName
The hostname of your outgoing mail server. Normally configured with the Servers Panel of the Preferences Dialogue.
newsagent.servers.SMTPPort
The port number of your outgoing mail server. Normally configured with the Servers Panel of the Preferences Dialogue.
newsagent.ui.LookAndFeel
The Swing look and feel to use. Will be supported in future versions of NewsAgent (not yet)
