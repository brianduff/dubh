Introduction

Setting Up the Relevance Agent

The Relevance Agent is an agent that lets you change the appearance of message headers based on how relevant you think they are.

To switch on the Relevance Agent, go to the List Agents Panel of the Preferences Dialogue (Edit:Preferences). Locate "Relevance Agent" in the list of available agents, and double click the list item to check the checkbox next to it. Click on the Configure... button at the bottom right of the panel to configure the behaviour of the agent.

Relevance Rules

The Relevance Agent uses a set of relevance rules to change your message headers. Each rule has a descriptive name and a set of associated newsgroups. When you first click on the Configure... button to configure the Relevance Agent, you will be presented with the Relevance Agent Configuration Dialogue. This dialogue has a list of all the relevance rules you have set up, and the newsgroups that each one affects. You can add, remove, edit or change the order of the rules from this dialogue.

Each rule is composed of a list of Scoring Conditions and Action Conditions.

Scoring Conditions

Each Relevance Rule has an associated set of scoring conditions. Each message header that passes through the agent starts with an initial score of zero. Scoring conditions alter that score, depending on the contents of the header fields. Some examples of scoring conditions are:

  • "If the From header contains Brian then subtract 100 from its score."
  • "If the Subject header starts with Re: then add 50 to its score."

As you can see, each scoring condition has four main configurable parts (in bold):

  • A header field to check (e.g. From, Subject)
  • How to compare the header (contains, starts with, ends with or equals)
  • Text to compare the header with (e.g. Brian, Re:)
  • A value to add or subtract from its score (e.g. +100, -50)

Scoring conditions are added to relevance rules using the Scoring Condition Editor. This dialogue is displayed when you use the Add / Edit Relevance Rule Wizard.

Action Conditions

Once a message header has been passed through all your scoring conditions, it will have a final score. The next step is to alter the message in some way (perform an action) based on that score. Some examples of action conditions are:

  • "If the score is less than -100 then delete the message."
  • "If the score is between -99 and 99 then display the message header in bold, red text."

There are two main configurable parts to action conditions:

  • A range and value(s) to compare the score to (less than x, equal to x, greater than x or between x and y)
  • An action to take if the condition is true (delete the message or change the font, size and colour of its header)

Action conditions are added to relevance rules using the Action Condition Editor. This dialogue is displayed when you use the Add / Edit Relevance Rule Wizard.
