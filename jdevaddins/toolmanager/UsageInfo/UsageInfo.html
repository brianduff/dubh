ToolManager Usage Information

ToolManager adds an Edit Tools... item into the IDE Tools menu which can be used to add menu and toolbar items for starting external programs from within the IDE.

Overview of this Tool

The Edit Tools dialog can be used to define tools which start external programs from within the IDE. You can customize where and how menu and toolbar items appear. You can choose a range of predefined values based on the current selection in the IDE to pass in as arguments to the tool or as the working directory.

This viewlet demonstrates the ToolManager in action.

Invocation instructions

This tool installs an Edit Tools... menu item into the Tools menu. Select this item to display the Edit Tools dialog.

Description of tool arguments

This section describes each of the tool arguments available in the Edit Tools dialog.

Default project sourcepath

The first entry in the sourcepath of the project associated with the currently selected item or active editor. This is the default location JDeveloper creates new classes in when using New Class...

Example: c:\oracle9i jdeveloper\jdev\mywork\SomeWorkspace\SomeProject\src
Macro Equivalent: {defaultSourcePath}

File name

The filename (without any path information) for the currently selected item or active editor.

Example: Widget.java
Macro Equivalent: {fileName}

File name without extension

The filename of the currently selected item or active editor with its extension stripped off.

Example: Widget
Macro Equivalent: {fileNameNoExtension}

File path

The full path of the currently selected item or active editor, including its filename.

Example: c:\oracle9i jdeveloper\jdev\mywork\SomeWorkspace\SomeProject\src\com\acme\Widget.java
Macro Equivalent: {path}

Full class name

The full class name of the currently selected item or active editor, including its package name. For this argument type to work properly, the selected item must be a Java or JSP file and in the sourcepath of a project.

Example: com.acme.Widget
Macro Equivalent: {fullClassName}

Full project sourcepath

The full sourcepath of the project associated with the currently selected item or active editor. This may include multiple directory entries even if your project sourcepath entry only has one entry because of libraries. If you just want the default sourcepath entry, use the default project sourcepath argument type.

Example: c:\oracle9i jdeveloper\jdev\mywork\SomeWorkspace\SomeProject\src;c:\oracle9i jdeveloper\BC4J\src\bc4jsrc.zip
Macro Equivalent: {sourcepath}

IDE install directory

The directory in which JDeveloper is installed.

Example: c:\oracle9i jdeveloper\jdev
Macro Equivalent: {ideInstallDir}

Package name

The package of the currently selected item or active editor. For this argument type to work properly, the selected item must be a Java or JSP file and in the sourcepath of a project.

Example: com.acme
Macro Equivalent: {packageName}

Project classpath

The full classpath of the project associated with the currently selected item or active editor. The classpath is an amalgamation of the classpaths of all libraries in use by the project and the output directory and additional classpath of the active project configuration.

Example: c:\simplestuff\classes;c:\libraries\acme.jar;c:\libraries\tools.jar
Macro Equivalent: {classpath}

Project HTML root

The HTML root of the project associated with the currently selected item or active editor.

Example: c:\oracle9i jdeveloper\jdev\mywork\SomeWorkspace\SomeProject\public_html
Macro Equivalent: {htmlRoot}

Project output directory

The output directory of the active configuration in the project associated with the currently selected item or active editor.

Example: c:\oracle9i jdeveloper\jdev\mywork\SomeWorkspace\SomeProject\classes
Macro Equivalent: {outputDirectory}

Prompt

If this argument type is included, a prompt dialog will appear when the tool is invoked. This dialog will contain a text field into which you can type a value for the argument.

Macro Equivalent: {prompt}

URL

The URL of the currently selected item or active editor. Be aware that some nodes in the IDE have URLs which use protocols that external programs may not understand.

Example: file:/C:/oracle9i jdeveloper/jdev/mywork/SomeWorkspace/SomeProject/src/com/acme/Widget.java
Macro Equivalent: {url}

 

Regular expressions for disabling tools

On the Enablement page of the edit tools dialog, you can enter a regular expression to only enable tools on files which match a certain regular expression. The full path of the file is matched against the regular expression. A description of the syntax accepted in this field is described here.

Some simple examples are below.

\.java$ matches all files ending in .java
(\.java$|\.jsp$|\.xml$) matches all files ending in .java, all files ending in .jsp and all files ending in .xml.
 acme matches all files with acme anywhere in their path or filename.
