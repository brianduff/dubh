TemplateMaker Usage Information

TemplateMaker provides the ability to define and use templates in the IDE. The templates can use the powerful features of the Jakarta Velocity template engine (see http://jakarta.apache.org/velocity).

Overview of this Tool

This tool installs a Save as Template... menu item into the File menu and the context menus of nodes in the navigator. You can use this menu item to create a template based on an existing file.

You can view, edit and delete templates in the Templates folder in the IDE system navigator.

The Object Gallery (accessible from New... in the File menu) can be used to create a new file based on a template. The tool adds a Templates category to the Object Gallery which contains all defined templates. When a template is selected, you are prompted for a path or package and filename. Alternatively, you can use the New From Template > context menu in the navigator.

TemplateMaker is integrated with Jakarta Velocity. This allows you to insert special macros and script into templates which are evaluated when a new file is created based on the template. More information on using velocity is available in the Velocity User Guide. Details on the IDE information you can obtain within velocity code in a template is available in the Context information for velocity section below.

Context information for velocity

The Velocity User Guide describes how velocity provides a way for templates to access properties and variables from within templates. JDeveloper provides several properties to the template engine when generating new files from templates. You can use these properties in your templates to automatically generate code based on information available at the time the template is used. For some examples of templates that use this information, see the sample templates section below.

The properties provided by the IDE to templates are as follows:

oracleide.destination.url

The URL of the output file, e.g. file:/C:/mywork/Test.java

oracleide.destination.path

The full local filesystem path of the output file, e.g. c:\mywork\Test.java

oracleide.destination.name

The filename of the output file, e.g. Test.java

oracleide.destination.nameWithoutExtension

The name of the output file, without its extension. This is useful for java files, since it's the name of the class. e.g. Test

oracleide.destination.packageName

The package name of the output file, assuming it's in a project with a valid sourcepath. e.g. org.acme.widgets

oracleide.template.url

The URL of the template file, e.g. file:/c:/jdeveloper/jdev/templates/Java Class.java

oracleide.template.path

The full local filesystem path of the template file, e.g. c:\jdeveloper\jdev\templates\Java Class.java

oracleide.template.name

The filename of the template file, e.g. Java Class.java

oracleide.template.nameWithoutExtension

The name of the template file, without its extension, e.g. Java Class

 

Sample templates

A number of sample templates are available for download. Each template contains comments describing how it works.

To use any of these templates, open it in the IDE and select Save as Template... from the File menu. The template should appear in the Object Gallery when you use File->New....
